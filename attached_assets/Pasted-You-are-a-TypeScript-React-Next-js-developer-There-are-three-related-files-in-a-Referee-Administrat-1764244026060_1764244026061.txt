You are a TypeScript React/Next.js developer.
There are three related files in a Referee Administration page:

pages/admin/referee/index.tsx (RefAdmin page)

components/admin/SectionHeader.tsx

components/admin/RefMatchFilter.tsx

Current behavior:

The RefAdmin page fetches matches and tournaments and maintains a filter state of type FilterState:

tournament: string

showUnassignedOnly: boolean

date_from?: string

date_to?: string

It calls fetchData(filter) whenever the filter state changes, which uses apiClient to call /matches and apply the filters via query params (date_from, date_to, tournament, assigned=false).

SectionHeader renders a RefMatchFilter and passes an onFilterChange callback but no filter object.

RefMatchFilter manages its own internal filter state (selectedTournament, showUnassignedOnly, dateRange) plus temporary “modal” state (tempSelectedTournament, tempShowUnassignedOnly, tempDateRange). It also fetches tournaments itself via apiClient.get('/tournaments').

What works:

Matches are filtered correctly in the backend via apiClient based on the filter state in index.tsx.

Reset in the filter modal clears filters and shows all matches again.

What does NOT work:

The funnel icon in RefMatchFilter does not stay solid when filters are applied; it often stays in the outline state.

When reopening the filter modal after clicking “Apply”, the previously chosen filter values are not consistently shown (the dialog does not always reflect the current filter).

The filter state in RefMatchFilter is not synchronized with the RefAdmin page filter state; the component holds its own applied and temp states, separate from the FilterState in index.tsx.

Tournaments are fetched twice (once in index.tsx and once inside RefMatchFilter) using apiClient.

filter: FilterState

onFilterChange: (filter: FilterState) => void

showUnassignedOnly: false
