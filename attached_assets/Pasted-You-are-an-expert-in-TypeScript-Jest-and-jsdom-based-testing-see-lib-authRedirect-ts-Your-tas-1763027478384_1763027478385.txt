How to structure the mocks
Please create small, explicit mocks inside the test file:
A minimal MockStorage implementing the Storage interface:
class MockStorage implements Storage {
  private store = new Map<string, string>();

  get length(): number {
    return this.store.size;
  }

  clear(): void {
    this.store.clear();
  }

  getItem(key: string): string | null {
    return this.store.has(key) ? this.store.get(key)! : null;
  }

  key(index: number): string | null {
    return Array.from(this.store.keys())[index] ?? null;
  }

  removeItem(key: string): void {
    this.store.delete(key);
  }

  setItem(key: string, value: string): void {
    this.store.set(key, value);
  }
}
In tests, use instances of MockStorage and assert on their internal state (e.g. getItem('access_token') is null after calling redirectToLogin).
